1) Consent Object

A Consent Object is a machine-readable record that represents who is giving consent, what they are consenting to, under what conditions, and for how long.

Required properties

  • subject_id
  • scope
  • use_case
  • boundaries
  • expiry
  • created_at
  • provenance

Behavioural guarantees

  • No identity-based generation without valid consent
  • Consent remains revocable at any time
  • Consent is not transferable unless explicitly stated
  • Absence of consent → fail-closed or fiction fallback

2) Identity Token

An Identity Token encodes the link between a human being and AI-generated content that depicts or imitates them.

Required properties

  • subject_id
  • token_id
  • issued_at
  • Reference to a valid Consent Object

System guarantees

  • Cannot be created without consent
  • Invalidated upon revocation
  • Cannot be used for impersonation or deepfakes

3) Revocation Signal

A Revocation Signal withdraws consent and disables identity use globally.

System obligations

  • Propagate revocation immediately
  • Prevent further identity-based generation
  • Invalidate associated Identity Tokens
  • Default to blocked or fictionalised output

4) Safety Mode Flag

A Safety Mode Flag determines which content types the system may generate.

Minimum required modes

  • MINOR_MODE
  • ADULT_MODE

MINOR_MODE guarantees

  • No erotic content
  • No minor sexualisation
  • Fail-closed on ambiguity

ADULT_MODE guarantees

  • Requires Consent Objects
  • Still prohibits minor/animal sexualisation
  • Provides revocation controls

5) Fictionalisation Engine

The Fictionalisation Engine replaces real identities with fully fictional characters when consent is absent or revoked.

Hard constraints

  • Must not recreate real individuals
  • Must not enable identity reverse-engineering
  • Must not mimic minors or specific real people
